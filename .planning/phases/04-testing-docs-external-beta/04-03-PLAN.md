**파일 1: `components.multiline.test.tsx` 기존 파일에 it() 블록 추가**

먼저 기존 파일을 읽어 이미 커버된 케이스를 파악한다. 누락된 케이스만 추가.

일반적으로 누락 가능한 케이스들:
- **Ctrl+J 개행**: `stdin.write('\x0a')` → onSubmit 호출 안 됨, 줄바꿈 누적
- **Ctrl+K 뒤 삭제**: 텍스트 'hello' 입력 후 Ctrl+A로 처음 이동 + Ctrl+K → 빈 문자열
- **Ctrl+W 단어 삭제**: 'hello world' 입력 후 Ctrl+W → 'hello '
- **Ctrl+U 전체 삭제**: 'hello' 입력 후 Ctrl+U → ''
- **↑/↓ 히스토리**: 히스토리가 비어있을 때 ↑ 키 입력 시 크래시 없음

각 케이스:
```typescript
it('Ctrl+J 는 개행만 하고 onSubmit 호출 안 함 (INPT-02)', () => {
  const onSubmit = vi.fn()
  const {stdin} = render(<MultilineInput onSubmit={onSubmit} />)
  stdin.write('hello')
  stdin.write('\x0a')  // Ctrl+J
  expect(onSubmit).not.toHaveBeenCalled()
})

it('Ctrl+U 전체 삭제 후 빈 입력 (INPT-04)', () => {
  const onSubmit = vi.fn()
  const {stdin, lastFrame} = render(<MultilineInput onSubmit={onSubmit} />)
  stdin.write('hello')
  stdin.write('\x15')  // Ctrl+U
  // 입력 버퍼가 비어있어야 함 — Enter 입력 시 빈 문자열로 submit
  stdin.write('\r')
  // onSubmit 호출되거나, 빈 입력이면 호출 안 될 수 있음 — 구현에 맞게 조정
  // 최소한 크래시 없음 검증
  expect(() => lastFrame()).not.toThrow()
})

it('빈 히스토리에서 ↑ 키 입력 시 크래시 없음 (INPT-03)', () => {
  const {stdin, lastFrame} = render(<MultilineInput onSubmit={vi.fn()} />)
  stdin.write('\x1b[A')  // ↑
  expect(() => lastFrame()).not.toThrow()
})
```

기존 파일 구조를 파악한 후 기존 describe 블록 내에 위 케이스들을 추가. 이미 있는 것은 중복 추가하지 않는다.
