# Harness 고도화 세션 핸드오프

## 완료된 작업

### 1차 고도화 (이전 세션)
- `session/compactor.py`: 압축 임계값 24KB → 12KB
- `config.py`: 환경변수 오버라이드 (HARNESS_OLLAMA_URL, HARNESS_MODEL, HARNESS_CTX, HARNESS_TEMP, HARNESS_PREDICT)
- `tools/fs.py`: `edit_file`에 `replace_all` 옵션 추가
- `tools/git.py`: `git_add`, `git_commit`, `git_checkout` 추가
- `tools/web.py`: `_clean_query` 완화 (30% 이상일 때만 제거)
- `tools/__init__.py`: 새 도구 3개 등록 (13→16개)

### 2차 고도화 (이번 세션)
- `config.py`: `runtime_override()` 함수 + `AGENT_TIMEOUT` 추가
- `profile.py`: model/ollama_url/temperature/num_ctx/num_predict DEFAULTS + 템플릿 [모델 설정] 섹션
- `main.py`: `config.runtime_override(profile)` 호출 (profile 로드 직후)
- `agent.py`: `import config`로 교체, 반성 루프 에러 상세화, 전체 타임아웃 추가
- `session/compactor.py`: `import config`로 교체, 요약 프롬프트 구조화 (## 완료된 작업 / ## 발견된 사실 / ## 현재 상태)
- `tools/shell.py`: stdout/stderr 8000자 초과 시 head+tail 트런케이션
- `tools/git.py`: `git_stash` 추가
- `tools/__init__.py`: `git_stash` 등록 (총 17개 도구)

**모델 전환**: `.harness.toml`에 `model = "llama3.3:70b"` 한 줄로 전환 가능

## 다음 논의 예정 주제

**자체 개선 기능 고도화** (`/improve`, `/learn`, `/evolve` 관련)
- `main.py`: `do_improve()`, `do_learn()`, `do_cplan()` 함수
- `tools/improve.py`: 소스 백업/검증/복원
- `evolution/`: engine.py, history.py, idle_runner.py, scorer.py, tracker.py
- `session/`: analyzer.py (세션 분석), compactor.py

### 현재 자체 개선 흐름
1. `/improve` → 백업 → agent.run(IMPROVE_SYSTEM 프롬프트) → 검증
2. `/learn` → 세션 요약 → HARNESS.md 갱신
3. `/evolve` → evolution 엔진 즉시 실행

### 알려진 한계 (분석 전)
- evolution/ 모듈 미검토 (engine.py, scorer.py 등)
- `/improve`가 실제로 무엇을 개선하는지, 품질 검증이 충분한지 불명확
- `/learn`의 HARNESS.md 갱신 품질 미검증

## 프로젝트 구조
```
~/harness/
├── main.py             # REPL + 슬래시 명령어
├── agent.py            # Ollama 에이전트 루프
├── config.py           # 설정 (env override + runtime_override)
├── profile.py          # 3단계 config merge (DEFAULTS → ~/.harness.toml → .harness.toml)
├── tools/
│   ├── __init__.py     # 17개 도구 정의
│   ├── fs.py, shell.py, git.py, web.py
│   └── improve.py      # 자체 개선용
├── session/
│   ├── compactor.py    # 세션 압축 (12KB 임계값)
│   └── analyzer.py
└── evolution/          # 미검토 — 다음 분석 대상
    ├── engine.py
    ├── scorer.py
    ├── history.py
    └── tracker.py
```
