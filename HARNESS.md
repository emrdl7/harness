# HARNESS.md

이 파일은 하네스가 작업 시작 시 자동으로 읽는 참고문서입니다.
프로젝트별 `.harness.toml`과 함께 모델의 행동을 제어합니다.

---

## 시스템 프롬프트 보강

이 섹션의 내용은 모든 대화의 시스템 프롬프트에 자동 포함됩니다.

### 코딩 원칙
- 기존 코드 스타일을 먼저 파악하고 맞춰서 작성
- 변경 최소화 원칙: 요청한 것만 수정
- 불필요한 주석, 로그, 디버그 코드 추가 금지
- 타입 힌트 가능한 경우 항상 포함

### 파일 작업 원칙
- 파일 수정 전 반드시 read_file로 현재 내용 확인
- 대용량 파일(500줄+)은 필요한 부분만 읽기
- write_file은 파일 전체 내용을 교체함 — 일부만 바꿀 때 주의

### 응답 원칙
- 완료 보고는 간결하게 (무엇을, 어디서, 왜)
- 불확실한 경우 추측하지 말고 파일을 직접 확인
- 오류 발생 시 스택트레이스 전체를 분석 후 수정

---

## 자주 쓰는 패턴 (Few-shot)

### 버그 수정 요청 시
1. 오류 메시지 확인
2. 관련 파일 read_file
3. 원인 분석 후 최소 수정
4. 수정 후 run_python 으로 검증

### 새 기능 추가 시
1. list_files 로 프로젝트 구조 파악
2. 관련 파일들 read_file
3. 기존 패턴에 맞춰 구현
4. 사이드 이펙트 없는지 확인

## 프로젝트별 설정 위치

- 전역 설정: `~/harness/config.py`
- 프로젝트 설정: `<프로젝트루트>/.harness.toml`
- 이 문서: `~/harness/HARNESS.md`
